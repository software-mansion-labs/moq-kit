## Build Commands

### iOS
```bash
./scripts/build-ios.sh           # Full build: XCFramework + Swift package
./scripts/build-xcframework.sh   # Rust → UniFFI → XCFramework only
swift build --package-path ios --sdk iphonesimulator --triple arm64-apple-ios16.0-simulator
```

### Android
```bash
./scripts/build-android.sh                    # Kotlin bindings + .so
cd android/moqkit && ./gradlew :MoQKit:manualBuild  # Assemble AAR + copy to examples
```

### Rust (vendor/moq)
```bash
# From moq-kit root — targets the vendor/moq workspace
cargo build --release --package moq-ffi --target aarch64-apple-ios
cargo build --release --package moq-ffi --target aarch64-apple-ios-sim

# Android
cargo ndk -t arm64-v8a build --release --package moq-ffi

# C bindings only (libmoq)
cargo build --release --package libmoq

# vendor/moq internal checks
cd vendor/moq && just check   # tests + linting
cd vendor/moq && just fix     # auto-format
```

Use `--profile release-with-debug` instead of `--release` to include DWARF debug symbols.

## Architecture

moq-kit wraps **moq-ffi** (Rust) with idiomatic Swift and Kotlin APIs. The stack:

```
moq-kit (Swift / Kotlin)    ← platform APIs in this repo
moq-ffi (UniFFI bindings)   ← vendor/moq/rs/moq-ffi/
libmoq (C bindings)         ← vendor/moq/rs/libmoq/ (pure C API)
hang (media layer)          ← codecs, containers, catalogs
moq-lite (transport)        ← pub/sub over QUIC
QUIC / WebTransport         ← network transport
```

**vendor/moq** is a git submodule pointing to `moq-dev/moq`. The UniFFI crate lives at `vendor/moq/rs/moq-ffi/`, the C bindings at `vendor/moq/rs/libmoq/`.

## UniFFI Bindings (critical)

Both platforms use **UniFFI** via the standalone `moq-ffi` crate (NOT libmoq):

- Rust uses `--package moq-ffi` (libmoq is now pure C bindings only)
- iOS: `build-xcframework.sh` runs `uniffi-bindgen`, generates `ios/Sources/MoQKit/moq.swift`
- Android: `build-android.sh` generates Kotlin under `android/moqkit/MoQKit/src/main/java/uniffi/moq/`
- **Never edit these generated files manually** — they're overwritten every build

Generated UniFFI types: `VideoConfig`, `AudioConfig`, `FrameData`, `AnnouncedInfo`, `MoqError`
Generated UniFFI protocols/callbacks: `SessionCallback`, `CatalogCallback`, `FrameCallback`, `AnnounceCallback`

Important field names:
- `VideoConfig.description: Data?` — codec init data (not `codecDescription`)
- `AudioConfig.sampleRate: UInt32`
- `FrameData`: `payload`, `timestampUs`, `keyframe`

## iOS SDK Structure (`ios/`)

- `Package.swift` — Swift package with binary target `moqFFI` (the XCFramework)
- `Sources/MoQKit/moq.swift` — auto-generated UniFFI bindings
- `Sources/MoQKit/MoQSession.swift` — `@MainActor` class managing connection lifecycle
- `Sources/MoQKit/MoQTransport.swift` — QUIC connection management
- `Sources/MoQKit/Catalog.swift` — track metadata (`MoQCatalog`, `MoQVideoTrack`, `MoQAudioTrack`)
- `Sources/MoQKit/MediaTrack.swift` — video/audio frame subscriptions via `AsyncStream`
- `Sources/MoQKit/SampleBufferFactory.swift` — `CMFormatDescription` / `CMSampleBuffer` builders
- `ios/Frameworks/libmoq.xcframework` — git-ignored, generated by build script

Callback wrappers (`TransportCB`, `AnnounceCB`, `CatalogCB`, `FrameCB`) are private helpers that bridge UniFFI callbacks → Swift `AsyncStream` continuations. Use `try?` on `close()` calls.

Error types: `MoQSessionError` (high-level), `MoqError` (low-level UniFFI, note lowercase 'q').

## Android SDK Structure (`android/moqkit/`)

- `MoQKit/build.gradle.kts` — AGP 9.0.1, Kotlin 2.0.21, Media3 1.9.2, minSdk 30
- `MoQKit/src/main/java/com/swmansion/moqkit/MoQSession.kt` — coroutine-based session manager
- `MoQKit/src/main/java/com/swmansion/moqkit/MoQMediaSource.kt` — custom `BaseMediaSource` bridging `Flow<FrameData>` → `SampleQueue` → ExoPlayer
- `MoQKit/src/main/java/com/swmansion/moqkit/MediaFactory.kt` — `MediaFormat` builders for video/audio
- `MoQKit/src/main/java/com/swmansion/moqkit/MediaTrack.kt` — `Flow<FrameData>` subscriptions
- `MoQKit/src/main/java/uniffi/moq/` — auto-generated UniFFI Kotlin bindings

Key patterns:
- `startTrack(context, videoIndex, audioIndex)` must be called on the main thread, returns `ExoPlayer?`
- `@file:OptIn(UnstableApi::class)` required in `MoQMediaSource.kt` and `MediaFactory.kt`
- State transitions use `compareAndSet()` for atomicity
- `media3-exoplayer` is an `api()` dependency (consumers get the `ExoPlayer` type)

## Key Concepts

- **Origin** — routing point; publishers and subscribers connect through the same origin
- **Session** — a QUIC connection to a MOQ relay with publish/consume origins
- **Broadcast** — named collection of tracks addressed by path
- **Track** — single media stream (one video rendition or audio track)
- **Group** — decodable unit starting with a keyframe; can be independently decoded for latency-based skipping
- **Catalog** — JSON track describing all other tracks (codec params, resolution, sample rate)
